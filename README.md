# [cite_start]Веб-платформа автошколы «LADA» с личным кабинетом [cite: 7]

[cite_start]Репозиторий содержит курсовой проект по разработке функциональной и адаптивной веб-платформы для автошколы «LADA»[cite: 52]. [cite_start]Платформа представляет собой единое цифровое пространство, объединяющее информационный сайт для привлечения клиентов и защищенную зону личных кабинетов для автоматизации учебного процесса[cite: 109, 111].

---

## 🛠 Технологический стек

[cite_start]Проект реализован с использованием классического, надежного и производительного стека технологий без тяжелых фреймворков, что обеспечивает высокую скорость загрузки страниц[cite: 90, 99, 105]:

* [cite_start]**Frontend:** HTML5 (семантическая разметка), CSS3 (Flexbox, Grid, `@media`-запросы для адаптивности), Native JavaScript (интерактивность интерфейса, валидация форм, динамическое переключение вкладок)[cite: 93, 96, 97, 98, 99].
* [cite_start]**Backend:** PHP (обработка запросов, авторизация, сессии, генерация динамического контента)[cite: 101].
* [cite_start]**База данных:** PhpMyAdmin / MySQL (реляционная БД с использованием безопасных параметризованных запросов PDO и хеширования паролей)[cite: 102, 103].
* [cite_start]**Среда разработки:** Visual Studio Code[cite: 91, 158].

---

## 👥 Ролевая модель и функционал

[cite_start]Система разделена на публичную часть и приватную зону (личные кабинеты) с четким разграничением прав доступа[cite: 58, 111]:

### 1. Публичный сайт (Для всех пользователей)
* [cite_start]**Главная страница:** Презентация автошколы, ее ценностей и преимуществ[cite: 123].
* [cite_start]**Каталог курсов:** Информация о программах обучения для категорий «А», «В», «C», «D», «A1», «M» с возможностью мгновенной подачи заявки[cite: 63, 123].
* [cite_start]**Интерактивный FAQ:** Ответы на часто задаваемые вопросы учеников[cite: 126].
* [cite_start]**Отзывы:** Блок с реальными историями выпускников автошколы[cite: 128].
* [cite_start]**Контакты:** Интеграция с интерактивной картой (Яндекс.Карты), информация о режиме работы и форма обратной связи[cite: 131, 132].

### 2. Личный кабинет ученика
* [cite_start]Просмотр текущего прогресса обучения (количество пройденных теоретических и практических занятий)[cite: 136].
* [cite_start]Динамическое расписание ближайших и только актуальных занятий[cite: 113, 136].
* [cite_start]История посещенных или отмененных занятий[cite: 113].
* [cite_start]Контроль статуса оплаты и отслеживание срока действия договора[cite: 136].

### 3. Личный кабинет инструктора
* [cite_start]Просмотр закрепленных учебных групп и детального списка учащихся[cite: 139].
* [cite_start]Управление расписанием: возможность добавления как групповых, так и индивидуальных занятий[cite: 139].
* [cite_start]Выставление отметок о посещаемости студентов[cite: 140].
* [cite_start]Просмотр личной статистики (всего занятий, проведено, общая посещаемость) и истории уроков[cite: 140].

### 4. Панель администратора (Максимальный доступ)
* [cite_start]**Дашборд:** Общая статистика системы (количество активных учеников, инструкторов и групп)[cite: 144].
* [cite_start]**CRM пользователей:** Полное управление (CRUD) аккаунтами учеников, инструкторов и админов[cite: 145].
* [cite_start]**Управление группами:** Создание, редактирование и удаление учебных групп[cite: 146].
* [cite_start]**Обработка заявок:** Менеджмент входящих заявок на обучение с сайта и перевод их в статус «Обработано»[cite: 147].
* [cite_start]**Система обращений:** Просмотр и фиксация ответов на сообщения из формы обратной связи[cite: 148].
* [cite_start]**Финансы и учет:** Мониторинг всех платежей, добавление новых транзакций и функция печати чека оплаты[cite: 149].

## 📂 Структура базы данных

[cite_start]Реляционная база данных включает в себя следующие связанные сущности[cite: 104]:
* [cite_start]`users` — учетные данные всех пользователей (хешированные пароли, роли: админ, инструктор, ученик)[cite: 102, 104].
* [cite_start]`groups` — учебные группы автошколы[cite: 104].
* [cite_start]`lessons` — групповые и индивидуальные занятия (теория/практика, место, время, статус)[cite: 104].
* [cite_start]`payments` — история транзакций и статус оплат учеников[cite: 104].
* [cite_start]`applications` — заявки на обучение с главной страницы[cite: 104, 147].
* [cite_start]`feedbacks` — обращения пользователей через форму контактов[cite: 104, 148].

## 🚀 Безопасность и оптимизация

1.  [cite_start]**Защита от SQL-инъекций:** Все SQL-запросы к базе данных используют механизм подготовленных выражений (параметризованные запросы PDO)[cite: 102].
2.  [cite_start]**Безопасность данных:** Пароли пользователей не хранятся в открытом виде, а подлежат обязательному криптографическому хешированию на стороне сервера[cite: 102].
3.  [cite_start]**Адаптивная верстка:** Интерфейс полностью оптимизирован под мобильные телефоны, планшеты и десктопы (принцип Mobile First)[cite: 97, 117].
4.  [cite_start]**Кроссбраузерность:** Код протестирован и стабильно работает в Google Chrome, Яндекс.Браузере, Safari и Firefox[cite: 118].
