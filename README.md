# Веб-платформа для автошколы “LADA” c личным кабинетом обучающегося

Курсовой проект по теме цифровизации образовательного процесса автошколы. Платформа позволяет привлекать новых клиентов через публичный сайт, а также автоматизировать обучение с помощью раздельных личных кабинетов для учеников, инструкторов и администратора[cite: 1].

### 🛠 Стек технологий:
* **Фронтенд:** HTML5, CSS3, JavaScript (адаптивная верстка, интерактивные элементы).
* **Бэкенд:** PHP (обработка сессий, авторизация, логика кабинетов).
* **База данных:** PhpMyAdmin / MySQL (реляционная БД, PDO-запросы).
* **Среда разработки:** Visual Studio Code.

## 💻 Обзор интерфейса и страниц проекта

### 1. Главная страница
Пользователь попадает сюда сразу после перехода на сайт. Здесь находится краткое описание автошколы, её преимуществ и кнопка перехода к курсам.
![Главная страница](<img width="974" height="443" alt="image" src="https://github.com/user-attachments/assets/36fe4fb2-22a0-45ee-9535-88a03d5d8968" />
)

### 2. Список курсов
Раздел для выбора программы обучения по различным категориям водительских прав. Из карточки курса можно отправить быструю заявку на обучение.
![Список курсов](папка_с_картинками/screenshot_2.png)

### 3. Раздел FAQ
Страница со списком часто задаваемых вопросов и раскрывающимися ответами на них[cite: 1].
![FAQ](папка_с_картинками/screenshot_3.png)

### 4. Отзывы учеников
Блок с реальными отзывами людей, которые уже завершили обучение в автошколе[cite: 1].
![Отзывы](папка_с_картинками/screenshot_4.png)

### 5. Страница контактов
Содержит адрес, телефон, email, график работы автошколы, встроенную интерактивную карту и форму обратной связи для вопросов[cite: 1].
![Контакты](папка_с_картинками/screenshot_5.png)

### 6. Вход в личный кабинет
Единая страница авторизации[cite: 1]. Система проверяет введенные Email и пароль, после чего автоматически определяет роль пользователя и открывает нужный кабинет[cite: 1].
![Вход](папка_с_картинками/screenshot_6.png)

### 7. Личный кабинет ученика
Интерфейс для учащегося, где отображается его текущая категория, статус обучения, прогресс по теории и практике, а также карточки ближайших занятий[cite: 1].
![Личный кабинет ученика](папка_с_картинками/screenshot_7.png)

### 8. Кабинет инструктора — Группы и занятия
Здесь инструктор может видеть список закрепленных за ним учебных групп и календарь запланированных теоретических или практических уроков[cite: 1].
![Группы и занятия](папка_с_картинками/screenshot_8.png)

### 9. Кабинет инструктора — Добавление занятия
Специальная форма, позволяющая инструктору быстро внести в систему новое групповое или индивидуальное занятие, указав дату, время, тему и место[cite: 1].
![Добавить занятие](папка_с_картинками/screenshot_9.png)

### 10. Кабинет инструктора — Статистика и ученики
Инструктор может отслеживать список своих учеников, общую посещаемость в процентах, количество проведенных уроков и историю занятий[cite: 1].
![Статистика инструктора](папка_с_картинками/screenshot_10.png)

### 11. Кабинет администратора — Статистика и пользователи
Главная панель управления администратора[cite: 1]. Показывает общее количество учеников, инструкторов и групп, а также содержит CRUD-таблицу для управления пользователями[cite: 1].
![Статистика и пользователи](папка_с_картинками/screenshot_11.png)

### 12. Кабинет администратора — Список групп
Панель для контроля учебных групп[cite: 1]. Администратор может добавлять новые группы, назначать им инструктора, менять статус («обучение» / «завершено») и даты[cite: 1].
![Список групп](папка_с_картинками/screenshot_12.png)

### 13. Кабинет администратора — Новые заявки
Сюда поступают все заявки от пользователей, заполнивших форму на главной странице[cite: 1]. После звонка клиенту администратор отмечает заявку как обработанную[cite: 1].
![Новые заявки](папка_с_картинками/screenshot_13.png)

### 14. Кабинет администратора — Новые обращения
Список сообщений, отправленных пользователями через страницу «Контакты»[cite: 1]. Администратор может ответить клиенту по телефону или email и перенести обращение в историю[cite: 1].
![Новые обращения](папка_с_картинками/screenshot_14.png)

### 15. Кабинет администратора — Платежи
Финансовый блок, где фиксируются все оплаты за обучение[cite: 1]. Доступны функции добавления платежа, выбора способа оплаты (онлайн, карта, перевод) и печати чека[cite: 1].
![Платежи](папка_с_картинками/screenshot_15.png)
